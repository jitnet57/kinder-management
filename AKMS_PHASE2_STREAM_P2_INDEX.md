## 각 LTO의 구조

### 표준 JSON 구조

```json
{
  "id": "domain_visual_lto01",
  "order": 1,
  "name": "색상 구분하기",
  "goal": "빨강, 파랑, 노랑, 초록 등 기본 색상 4개를 정확히 구분",
  "stoCount": 4,
  "stos": [
    {
      "order": 1,
      "name": "색상 구분하기 - 보조 있이 시도",
      "description": "성인의 신체적/언어적 보조로 색상 구분하기 시도"
    },
    { ... STO 2, 3, 4 ... }
  ],
  "teachingTips": {
    "prompt_hierarchy": "1. 모델링... → 2. 신체적 보조... → 3. 제스처 힌트 → 4. 색상명 힌트 → 5. 독립적 반응",
    "reinforcement": "정답 후 즉시 강화(칭찬: '맞아!' + 선호하는 활동 5-10초)",
    "error_correction": "오류 시 즉시 모델링으로 보여주고 3초 내 아동의 모방 유도",
    "generalization": "매 세션마다 다른 색상 물건 사용(블록, 그림카드, 실제 물건)",
    "data_collection": "시도 횟수, 보조 수준(full/partial/none), 정답률 기록",
    "prerequisite": "주의집중 5초 이상, 성인의 제스처 모방 능력",
    "motivation": "아동이 좋아하는 색깔 강화물 사용",
    "family_involvement": "부모에게 일상물(우유팩, 신호등, 과일)로 색상 구분 연습 가이드 제공"
  }
}
```

### 핵심 요소

1. **STO (Support To Independence) 4단계**
   - Stage 1: 보조 있이 시도 (Full Support)
   - Stage 2: 부분적 독립 (Partial Independence)
   - Stage 3: 독립적 수행 (Independence)
   - Stage 4: 일반화 (Generalization)

2. **TeachingTips 8개 항목**
   - `prompt_hierarchy`: 단계적 보조 전략
   - `reinforcement`: 강화 방법
   - `error_correction`: 오류 교정
   - `generalization`: 일반화 전략
   - `data_collection`: 자료 수집
   - `prerequisite`: 선행 기술
   - `motivation`: 동기 유발
   - `family_involvement`: 가정 연계
