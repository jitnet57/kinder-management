[이벤트 발생 시점 및 트리거]

   - CurriculumContext.completeSessionTask() 호출 시
   - 메타데이터: childId, ltoId, ltoName, domainName, score, completedAt

2. 점수 향상 (score_improved)
   - SessionTask 점수 업데이트 시 (이전 점수보다 높음)
   - 메타데이터: childId, ltoId, previousScore, score, improvementPercent

3. 메시지 수신 (message_received)
   - Message 생성 시
   - 메타데이터: childId, messageId, conversationId, senderName, messagePreview

4. 피드백 수신 (feedback_received)
   - Feedback 생성 시
   - 메타데이터: childId, conversationId, senderName

5. 마일스톤 달성 (milestone_achieved)
   - LTO 완료 수, 영역 숙달 등 특정 조건 충족 시
   - 메타데이터: childId, milestoneId, milestoneType, celebrationMessage

6. 승인 필요 (approval_required)
   - 관리자 승인 필요한 항목 생성 시
   - 메타데이터: childId, ltoId, ltoName

7. 세션 예약 (session_scheduled)
   - Schedule 생성 시
   - 메타데이터: childId

8. 리마인더 (reminder)
   - 특정 조건 또는 일정 시간 도래 시
   - 메타데이터: childId, messagePreview

[주요 기능]

1. 실시간 필터링
   - 알림 타입: 8가지 이벤트
   - 심각도: info, success, warning, urgent
   - 우선순위: normal, high, urgent
   - 아동별 필터
   - 읽음 상태: all, read, unread
   - 날짜 범위

2. 정렬 옵션
   - 최신순 (기본)
   - 우선순위순

3. 조용한 시간 (Quiet Hours)
   - 설정된 시간대에 푸시 알림 자동 억제
   - 앱 내 알림은 계속 표시
   - 기본값: 22:00 ~ 08:00

4. 음소거 (Mute)
   - 특정 알림 타입 음소거 가능
   - 채널별 비활성화 가능

5. 알림 액션
   - 클릭 시 관련 페이지로 이동
   - 각 이벤트별 맞춤 액션 링크

6. 배치 처리
   - 동일 시간대 알림 통합
   - 푸시 알림 수 최적화

7. 자동 만료
   - 7일 이상 된 알림 자동 삭제

[사용 예제]

기본 사용:
  const { emitEvent } = useNotification();
  await emitEvent('lto_completed', {
    childId: 1,
    ltoId: 'lto_001',
    ltoName: 'Listen to 2-word utterances',
    domainName: 'Mand',
    score: 92,
    completedAt: new Date().toISOString(),
  });

고급 필터링:
  const { getNotifications } = useNotification();
  const notifications = getNotifications(userId, {
    priority: ['urgent', 'high'],
    childId: 1,
    readStatus: 'unread',
  });

실시간 리스닝:
  const { onNotificationReceived } = useNotification();
  const unsubscribe = onNotificationReceived((notification) => {
    console.log('새 알림:', notification);
  });
