# AKMS 사용 방법 가이드

## AKMS란?
**AKMS** = **A**BA **C**hild **M**anagement **S**ystem

유아 ABA(응용행동분석) 데이터를 체계적으로 관리하고 분석하는 웹 애플리케이션입니다.

---

## 시작하기

### 웹 접속
📱 **배포 URL**: https://aba-child.pages.dev  
🌐 **지원 브라우저**: Chrome, Safari, Firefox, Edge (모두 지원)  
📱 **반응형 지원**: PC, 태블릿, 모바일 모두 최적화

## 주요 메뉴 설명

### 1️⃣ 대시보드 (Dashboard)
홈페이지에서 전체 현황을 한눈에 파악하는 공간입니다.

#### 주요 요소
```
📊 통계 카드 (4개)
├─ 👧 등록된 아동: 현재 등록된 아동의 수
├─ 📝 이번 주 세션: 이번 주에 진행된 세션 횟수
├─ ✅ 완료율: 목표 달성률 (백분율)
└─ 📚 활성 커리큘럼: 현재 진행 중인 학습 목표 수
```

#### 통계 카드 활용
- **마우스 올리기 (호버)**: 색상이 진해집니다
- **상세보기 버튼**: 해당 통계의 상세 정보를 모달창으로 표시
- **내보내기 버튼**: 데이터를 JSON 파일로 다운로드

#### 성과 분석 그래프
- **차트 타입 선택**: 추세, 막대, 원형, 산점도 등 8가지 제공
- **시간 범위**: 7일, 30일, 90일, 365일 선택 가능
- **그룹화**: 일별, 주별, 월별, 아동별, 발달영역별로 분류

---

### 2️⃣ 스케줄 (Schedule)
한 주간의 모든 세션을 시각적으로 관리합니다.

#### 레이아웃
```
        월  화  수  목  금  토
오전    [ ][ ][ ][ ][ ][ ]
8-10시

오전    [ ][ ][ ][ ][ ][ ]
10-12시

오후    [ ][ ][ ][ ][ ][ ]
2-4시

오후    [ ][ ][ ][ ][ ][ ]
4-6시
```

#### 세션 추가하기
1. **+ 버튼 클릭**: 원하는 요일과 시간대 선택
2. **아동 선택**: 드롭다운에서 아동 이름 선택
3. **세션명 입력**: 예) "발음 연습", "색상 구분" 등
4. **추가 완료**: 세션이 컬러풀한 카드로 표시됨

#### 세션 관리
- **수정 (Edit)**: 세션명 변경 가능
- **삭제 (Delete)**: 세션 제거
- **색상 구분**: 각 아동마다 다른 색상으로 표시

💡 **팁**: 호버하면 아이콘이 색상이 변해서 상호작용 가능한 요소를 알 수 있습니다.

---

### 3️⃣ 아동정보 (Children)
모든 아동의 정보를 카드 형태로 관리합니다.

#### 아동 정보 입력
```
├─ 이름: 아동의 이름
├─ 생년월일: YYYY-MM-DD 형식
├─ 전화: 보호자 연락처
├─ 주소: 거주 지역 (지번주소)
└─ 기타정보: 발달 특성, 주의사항 등
```

#### 아동 카드 활용
- **아동 카드 클릭**: 상세 정보 및 과제 기록 조회
- **아바타**: SVG로 자동 생성 (초성 + 색상)
- **수정 버튼**: 아동 정보 수정
- **삭제 버튼**: 아동 제거
- **검색**: 아동 이름으로 빠른 검색

#### 아동 상세 조회
클릭하면 해당 아동의:
- 📋 과제 카드 (점수, 날짜, 비고)
- 📈 성과 그래프
- ✏️ 과제 수정/삭제 기능

---

### 4️⃣ 데이터 기록 (SessionLog)
매 세션마다 아동의 성과를 기록합니다.

#### 기록 프로세스
```
1️⃣ 날짜 선택
   └─ 기록할 세션 날짜 선택

2️⃣ 아동 선택
   └─ 드롭다운에서 아동 이름 선택

3️⃣ 커리큘럼 선택
   ├─ 발달영역 (언어, 인지, 운동 등)
   ├─ LTO (Long Term Objective)
   └─ STO (Short Term Objective)

4️⃣ 데이터 입력
   ├─ 점수: 0-100점 (슬라이더 또는 숫자)
   ├─ 시작/종료 시간
   └─ 비고: 관찰 내용, 특이사항

5️⃣ 완료 표시
   └─ "완료" 버튼 클릭 → 완료목록으로 이동
```

#### 각 과제의 기능
- **그래프 버튼**: 해당 과제의 성과 추이 그래프 표시
- **수정 버튼**: 입력한 데이터 수정
- **삭제 버튼**: 미완료 과제 삭제
- **완료 버튼**: 진행 중인 과제를 완료 처리

💡 **팁**: 점수는 0-100점 범위이며, 슬라이더로 빠르게 입력할 수 있습니다.

---

### 5️⃣ 완료목록 (Completion)
완료된 모든 세션의 기록을 관리합니다.

#### 필터링 옵션
- **아동 선택**: 특정 아동의 기록만 표시
- **시간 범위**: 이번 주, 지난 주, 이번 달, 전체
- **정렬**: 최신순, 오래된순

#### 통계 정보
- 📊 **이번 주 완료수**: 일주일간 완료된 세션
- 📋 **총 완료수**: 누적 완료 세션
- ⭐ **평균 점수**: 모든 완료 세션의 평균 점수

#### 테이블 정보
```
상태 | 아동 | 영역 | 과제 | 점수 | 완료시간
✅   |민준 |언어 |발음 |90점 |2026.04.20
```

---

### 6️⃣ 커리큘럼 (Curriculum)
학습 목표를 계층적으로 관리합니다.

#### 3단계 계층 구조
```
📚 발달영역 (Domain)
└─ 📖 LTO (Long Term Objective)
   └─ ✓ STO (Short Term Objective)
```

#### 예시
```
언어발달 (Domain)
├─ 발음 능력 개발 (LTO)
│  ├─ 기본 자음 발음 (STO)
│  ├─ 기본 모음 발음 (STO)
│  └─ 어휘 확장 (STO)
└─ 이해 능력 증진 (LTO)
   ├─ 지시 이해 (STO)
   └─ 질문 이해 (STO)
```

#### 커리큘럼 관리
- **발달영역 추가**: + 버튼으로 새 영역 생성
- **LTO 추가**: 각 발달영역 아래에 목표 추가
- **STO 추가**: LTO 아래에 세부 목표 추가
- **수정/삭제**: Edit/Delete 버튼으로 관리

💡 **팁**: 각 레벨의 항목을 마우스로 가리키면 Edit, Delete 버튼이 나타납니다.

### 7️⃣ 보고서 (Reports)
아동의 성과를 다양한 형태로 분석합니다.

#### 두 가지 보고서 타입

#### A. 👤 아동별 보고서
특정 아동의 상세 분석 정보를 제공합니다.

**선택 단계:**
1. 보고서 타입에서 "아동별 보고서" 클릭
2. 아동 선택 (4명 중 선택)
3. 자동으로 데이터 로드

**제공 데이터:**
```
📊 통계 카드
├─ 총 세션: 진행한 세션 횟수
├─ 평균 점수: 평균 성과 점수
└─ 개선도: 초기 대비 현재의 향상율

📈 점수 추이 그래프
└─ 시간에 따른 점수 변화 시각화

🎯 발달영역별 점수
└─ 각 영역의 평균 점수 비교

📋 상세 테이블
└─ 발달영역별 세션 수와 평균 점수
```

#### B. 📈 전체 통계
모든 아동의 종합 데이터를 비교합니다.

**제공 데이터:**
```
📊 전체 요약
├─ 등록 아동: 몇 명이 등록되어 있는가
├─ 총 세션: 전체 진행한 세션 횟수
└─ 평균 점수: 모든 세션의 평균 점수

👥 아동별 성과 비교
└─ 막대 그래프로 아동별 평균 점수 비교

🎯 발달영역별 분석
├─ 발달영역별 평균 점수
└─ 각 영역이 얼마나 많이 진행되었는지 표시

📊 점수 분포
└─ 점수 범위별 아동 수 (우수, 양호, 보통, 미흡)

📋 상세 통계 테이블
├─ 발달영역별 통계
└─ 아동별 상세 정보
```

#### 데이터 내보내기
- **내보내기 버튼**: 모달 또는 카드의 "내보내기" 버튼 클릭
- **형식**: JSON
- **파일명**: `report-[아동명]-[날짜].json`
- **용도**: 엑셀, 파워포인트 등에서 추가 분석 가능

## 자주 묻는 질문 (FAQ)

### Q1. 데이터는 저장되나요?
**A**: 모든 데이터는 브라우저의 로컬 스토리지에 저장됩니다. 브라우저 캐시를 지우면 데이터가 초기화됩니다.

### Q2. 여러 기기에서 같은 데이터를 볼 수 있나요?
**A**: 현재는 각 기기별로 독립적인 데이터를 가지고 있습니다. 향후 클라우드 연동 기능이 추가될 예정입니다.

### Q3. 세션을 수정했는데 변경이 반영되지 않았어요.
**A**: 페이지를 새로고침(F5) 해보세요. 혹은 저장 버튼을 다시 클릭하세요.

### Q4. 완료한 세션을 다시 수정할 수 있나요?
**A**: 완료된 세션은 "완료목록"에서 확인할 수 있지만, 수정하려면 데이터 기록에서 새로운 기록으로 추가해야 합니다.

### Q5. 그래프가 안 나와요.
**A**: 데이터 기록이 충분하지 않을 수 있습니다. 최소 2개 이상의 기록이 필요합니다.

### Q6. 아동을 삭제하면 과제 기록도 삭제되나요?
**A**: 현재는 아동 삭제 시 과제 기록은 유지됩니다. 필요시 수동으로 기록을 삭제해주세요.

## 팁 & 베스트 프랙티스

### 1. 효율적인 데이터 기록
```
✅ DO
- 세션 직후 바로 점수와 비고 기록
- 구체적인 관찰 내용 작성
- 일관된 시간에 기록

❌ DON'T
- 며칠 뒤에 기억으로 기록
- 점수만 입력하고 비고 생략
- 부정확한 날짜 입력
```

### 2. 커리큘럼 구성
```
✅ DO
- 명확한 계층 구조 유지
- STO는 구체적이고 측정 가능하게
- 정기적으로 커리큘럼 검토

❌ DON'T
- 4단계 이상 깊이의 계층 생성
- 모호한 목표 표현
- 커리큘럼을 방치하기
```

### 3. 스케줄 관리
```
✅ DO
- 한 주 단위로 미리 계획
- 아동의 능력 수준 고려
- 정기적으로 스케줄 검토

❌ DON'T
- 마지막 순간에 추가
- 과도한 세션 구성
- 스케줄을 수정하지 않기
```

### 4. 보고서 분석
```
✅ DO
- 정기적으로 보고서 생성 (주 1회)
- 아동별 진도 추적
- 개선도를 통해 효과 검증

❌ DON'T
- 단순히 점수만 보기
- 장기적 추이 무시
- 개선도 낮은 영역 방치
```

## 문제 해결

### 문제: 페이지가 안 로드됨
**해결책:**
1. 인터넷 연결 확인
2. 브라우저 새로고침 (Ctrl+F5)
3. 브라우저 캐시 삭제

### 문제: 데이터가 안 저장됨
**해결책:**
1. 저장 버튼 다시 클릭
2. 필수 항목이 모두 입력되었는지 확인
3. 브라우저 콘솔 확인 (F12)

### 문제: 그래프가 안 나타남
**해결책:**
1. 데이터 입력 확인 (최소 2개 필요)
2. 정렬 옵션 변경
3. 시간 범위 확인

### 문제: 호버 효과가 안 보임
**해결책:**
1. 마우스 움직임 확인
2. 터치 기기인 경우 탭 진행
3. 브라우저 업데이트
