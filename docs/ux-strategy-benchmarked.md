# Kinder ABA 관리 시스템 | UX 전략 재정의
## Design & Content Strategy Based on Benchmarking

**작성일**: 2026-04-26  
**Status**: 벤치마킹 분석 완료 → 전략 재정의  
**Reference Sites**: The Talcott Center (design), Inclusive ABA (content)

---

## 1. 핵심 인사이트 (Benchmarking)

### 1.1 The Talcott Center | 디자인 시스템
**성공 요소:**
- ✅ **계층적 네비게이션**: 부모의 인지 부하 최소화
- ✅ **신뢰 구축**: 자격증, 운영 기간(20+년), 팀 규모(60명) 즉시 노출
- ✅ **Pain Point 해결**: Fast-Track Intake → "진행 상황 손실 걱정?"에 직접 대응
- ✅ **감정적 공감**: 부모 인용문 + 지역 명시 → 커뮤니티 신뢰
- ✅ **임상 권위 + 온정성**: 보험 로고, 팀 경력 vs. 고객 후기

**우리 프로젝트 적용:**
- 아동 색상 자동 할당 + 시각적 신뢰 (이미 구현)
- 네비게이션을 더 계층적으로 리뉴얼 (대시보드 → 빠른 접근)
- 통계/신뢰 신호 추가 (완료율, 기록 일수, 커리큘럼 진행도)
- 교사/부모용 고객 후기 섹션

### 1.2 Inclusive ABA | 콘텐츠 전략
**성공 요소:**
- ✅ **다중 이해관계자**: 부모, 직원, 임상가를 위한 별도 진입점
- ✅ **결과 중심 메시징**: 전문 용어 회피 → "아이가 성공하도록 돕기"
- ✅ **형평성 프레임**: "어떤 아이도 뒤처지지 않아야 함"
- ✅ **점진적 참여 모델**: 연락 → 평가 → 승인 → 서비스
- ✅ **접근성**: 이중언어, 보험 처리, 집에서 편의

**우리 프로젝트 적용:**
- 역할별 네비게이션 (관리자/임상가/부모 보기 분리)
- "아동이 성장한다" 중심 메시징 (점수보다는 진행도)
- 보험 정보, 파일 처리, 온라인 기록 강조
- 장애 접근성 개선 (키보드 네비, 스크린리더, 명확한 라벨)

## 2. 재정의된 UX 전략

### 2.1 네비게이션 재구조화

**현재 (심플):**
```
Side Nav:
- 대시보드
- 스케줄
- 아동정보
- 데이터기록
- 완료목록
- 커리큘럼
```

**개선 (계층적):**
```
Top Bar: 로고 + 빠른 통계 (오늘 세션 수, 진행도) + 사용자 프로필

Side Nav:
├─ 📊 대시보드
│  └ Quick Actions (+ 새 아동, + 세션 기록, 보고서)
├─ 📅 스케줄
│  └ 주간 뷰, 월간 뷰, 충돌 경고
├─ 👶 아동 관리
│  ├─ 목록 뷰 (빠른 검색)
│  └─ 상세 프로필 (파일, 진행도, 히스토리)
├─ 📊 성과 기록
│  ├─ 일일 기록 (빠른 입력)
│  └─ 분석 (7일, 30일, 맞춤 기간)
├─ ✅ 완료 추적
│  └─ 필터 (기간, 아동, 상태)
└─ 🎓 커리큘럼
   ├─ 계층 뷰 (발달영역 → LTO → STO)
   └─ 할당 및 진행도
```

### 2.3 콘텐츠 필라 (Content Pillars)

**1. 결과 중심 메시징**
- ❌ "세션 점수 기록" → ✅ "아동의 성장 추적"
- ❌ "STO 완료" → ✅ "새로운 기술 습득"
- ❌ "커리큘럼 할당" → ✅ "개발 목표 설정"

**2. 형평성 & 접근성**
- 다국어 지원 (영어, 한국어 필수)
- 스크린리더 친화 (WCAG 2.1 AA 준수)
- 직관적 인터페이스 (전문 용어 최소화)
- 오프라인 기능 (WiFi 없을 때 로컬 저장)

**3. 다중 이해관계자**
- **관리자 보기**: 전체 기관 통계, 팀 관리, 보고서
- **임상가 보기**: 아동별 상세 기록, 실시간 입력, 그래프
- **부모 보기** (향후): 아동 진행도, 월별 보고서, 읽기 전용

### 2.4 점진적 참여 모델

**초기 설정:**
```
1단계: 로그인 → 계정 생성 (이메일, 비밀번호)
2단계: 기관 정보 → 이름, 위치, ABA 자격증
3단계: 첫 아동 추가 → 이름, 생년월일, 담당자
4단계: 커리큘럼 할당 → 발달영역 선택, 목표 설정
5단계: 첫 세션 기록 → 간단한 점수 + 메모
```

각 단계에서 **"계속" 버튼 하나** → 인지 부하 최소화

## 3. 시각적 재설계

### 3.1 색상 재평가

**현재 파스텔톤 유지, 강조점 조정:**

```
Primary (신뢰, 활동): 
- Pastel Teal (#A8D8D8) → 주요 버튼, 활성 상태

Secondary (성공, 성장):
- Pastel Green (추가) → 완료, 진행도 표시

Accent (주의, 경고):
- Pastel Pink (#F7B8C8) → 경고, 마감

Neutral (배경, 텍스트):
- 흰색 투명도 + 글래스 효과 유지
- 어두운 텍스트 (opacity 0.8 이상 = 접근성)

Child Colors (8가지 유지):
- Pastel Purple, Teal, Pink, ...
- 아동 식별용, 그래프 색상, 카드 배경
```

### 3.2 글래스 모르피즘 세부화

**유지할 점:**
- 반투명 배경 (rgba 0.7-0.8)
- 10-20px 블러 효과
- 1px 흰색 테두리

**개선점:**
- 카드에 **미묘한 그림자** (0.5px, 흑색 opacity 0.1)
  → 뎁스감 추가, 접근성 해침 없이
- 호버 시 **밝기 +5%** (그림자 강화, blur 유지)
- 포커스 상태 **명확한 테두리** (2px, teal)
  → 키보드 네비 사용자 보호

## 5. 접근성 개선 체크리스트

- [ ] WCAG 2.1 AA 준수 확인 (자동 스캔)
- [ ] 명도비 4.5:1 이상 (텍스트 대 배경)
- [ ] 키보드 전체 네비 가능 (Tab 키)
- [ ] 포커스 표시기 명확 (2px 테두리, 색상 대비)
- [ ] Alt 텍스트 모든 이미지
- [ ] 폼 라벨 <label> 연결
- [ ] 스크린리더 aria-label 추가
- [ ] 링크/버튼 목적 명확 (기술적 이름 아님)
- [ ] 색상만으로 정보 전달 금지 (텍스트 + 아이콘)
- [ ] 오프라인 기능 (LocalStorage 캐싱)

## 6. 측정 메트릭 (KPI)

**기능성 (Functionality):**
- 세션 기록 입력 시간 < 1분 (목표: 30초)
- 페이지 로드 < 300ms (목표: 200ms)
- 에러율 < 0.1% (uptime 99.9%)

**사용성 (Usability):**
- 사용자 만족도 4.5/5 이상
- 태스크 완료율 95% 이상
- 키보드/스크린리더 사용 가능률 100%

**영향 (Impact):**
- 월 활성 사용자 증가
- 보고서 생성 시간 60% 단축
- 부모 만족도 4.8/5 (향후 모바일 앱)

## 8. 결론

**세 가지 핵심 원칙:**
1. **신뢰**: 계층적 네비 + 신뢰 신호 + 명확한 자격증
2. **접근성**: 다국어 + 키보드 + 스크린리더 + 명도비
3. **성장**: 결과 중심 메시징 + 시각적 진행도 + 아동 중심

이 전략은 Talcott Center의 **임상 권위 + 온정성**과 Inclusive ABA의 **형평성 + 결과 중심** 원칙을 결합합니다.
