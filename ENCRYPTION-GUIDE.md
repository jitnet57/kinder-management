# 🔐 데이터 암호화 가이드

**목표**: USB에 저장된 백업 파일도 암호화하여 보안 강화  
**적용**: 브라우저 캐시 + 백업 파일 암호화  
**기술**: AES-256-GCM 암호화

---

## 📋 개요

### 1. 브라우저 캐시 암호화 (자동 저장)

```
사용자 작업 데이터
     ↓
 브라우저 메모리
     ↓
자동 저장 (비밀번호로 암호화)
     ↓
localStorage/IndexedDB
```

**특징**:
- ✅ 앱 종료 후에도 복구 가능
- ✅ 오프라인에서도 작동
- ✅ 매 변경사항 자동 저장

### 2. 백업 파일 암호화 (USB 전송)

```
PostgreSQL 덤프
    ↓
tar.gz 압축
    ↓
AES-256-GCM 암호화 (선택사항)
    ↓
USB에 저장
```

**특징**:
- ✅ USB를 분실해도 안전
- ✅ 사무실과 집 컴퓨터 간 안전한 전송
- ✅ 메타데이터 포함 (암호화 방식, 타임스탐프)

## 🔧 설정

### 브라우저 캐시 암호화

#### Step 1: 첫 실행 시 비밀번호 설정

```
[앱 실행]
    ↓
🔐 비밀번호 설정 모달
[새 비밀번호 입력] → [확인]
    ↓
✅ 캐시 암호화 활성화
```

#### Step 2: 비밀번호 요구사항

```
✅ 최소 8자 이상
✅ 대문자 포함 (A-Z)
✅ 소문자 포함 (a-z)
✅ 숫자 포함 (0-9)
✅ 특수문자 포함 (!@#$%^&*)
```

**강도 표시**:
- 🔴 약함 (1-2개 요구사항)
- 🟡 보통 (3개 요구사항)
- 🔵 좋음 (4개 요구사항)
- 🟢 매우 강함 (5개 모두)

#### Step 3: 비밀번호 관리

```bash
# 설정된 비밀번호 변경
Settings → 보안 → 비밀번호 변경

# 비밀번호 비활성화
Settings → 보안 → 보호 해제
```

---

### 백업 파일 암호화

#### 환경 변수 설정 (.env.local)

```bash
# 백업 암호화 활성화
BACKUP_ENCRYPTION=true

# 암호화 비밀번호 (자동 백업용)
# ⚠️ 매우 강력한 비밀번호 권장
BACKUP_PASSWORD=YourStrongPassword123!@#
```

#### 설정 확인

```bash
# 백업 로그에서 암호화 상태 확인
2026-04-26 00:00:00 ✅ 백업 작업 완료
2026-04-26 00:00:00 🔐 암호화: 활성화

# 또는
curl http://localhost:3000/api/admin/backup/status \
  -H "Authorization: Bearer YOUR_API_KEY" \
  | grep encrypted
```

## 🔄 사용 흐름

### 시나리오 1: 집 → 회사 (암호화된 백업)

```
집 컴퓨터 (22:00)
├─ 자동 백업 생성
│  └─ kinder-backup-2026-04-26.tar.gz (압축됨)
└─ 암호화 적용
   └─ kinder-backup-2026-04-26.tar.gz.enc ← USB에 복사

회사 컴퓨터 (09:00)
├─ USB 삽입
├─ API 호출: POST /api/sync/restore
│  ├─ Form Data:
│  │  ├─ backup: @kinder-backup-2026-04-26.tar.gz.enc
│  │  └─ password: (백업 암호화 비밀번호)
│  └─ 자동 복호화 + 복원
└─ 응답: ✅ 복원 완료 (14.56초)
   └─ 🔐 암호화된 파일
```

### 시나리오 2: 수동 백업 (실시간 동기화)

```
회사 컴퓨터 (17:00)
├─ API /api/admin/backup/trigger
│  └─ 즉시 백업 생성
├─ 암호화: BACKUP_PASSWORD 사용
└─ kinder-backup-2026-04-26.tar.gz.enc ← USB에 복사

집 컴퓨터 (저녁)
├─ USB 삽입
├─ 앱 복원 UI에서:
│  ├─ [파일 선택] → kinder-backup-2026-04-26.tar.gz.enc
│  ├─ [비밀번호 입력] → ●●●●●●●●
│  └─ [복원]
└─ ✅ 복원 완료
```

## 🛡️ 보안 고려사항

### 1. 비밀번호 관리

#### 강력한 비밀번호 작성

```
❌ 약함: password123, admin@123, Birthday!
✅ 강함: Kinder@2026%ABA#Secure, C0mpl3x!P@ssw0rd
```

#### 비밀번호 저장

```
✅ 권장:
  - 안전한 비밀번호 관리자 사용 (1Password, KeePass)
  - 종이에 암호화하여 금고에 보관
  - 신뢰할 수 있는 동료와 공유 (필요시)

❌ 금지:
  - 텍스트 파일에 저장
  - 이메일로 공유
  - 스티커에 붙여놓기
  - 동료들이 보기 쉬운 곳
```

### 2. 파일 보안

#### USB 드라이브 관리

```
물리적 보안:
├─ USB 분실 방지 (목걸이, 파우치)
├─ 정기적으로 위치 확인
└─ 사무실/집에서 안전한 장소에 보관

데이터 보안:
├─ 파일이 암호화되어 있으므로 안전
├─ 정기적으로 아카이브 백업 생성
└─ 분기마다 외장 HDD에 보관
```

#### 오래된 백업 삭제

```
로컬 저장소 (C:\kinder-backups\daily\):
├─ 자동 삭제: 30일 이상 된 파일
└─ 용량: 약 1.5GB (30개 × 50MB)

USB 보관:
├─ 3개월 이상 된 파일 수동 삭제
├─ 매월 1일 정리 작업
└─ 중요 백업은 아카이브로 별도 보관
```

### 3. 암호화 키 관리

#### PBKDF2 (Password-Based Key Derivation Function 2)

```
사용자 비밀번호
    ↓
+ 랜덤 Salt (16바이트)
    ↓
100,000 iterations (SHA-256)
    ↓
AES-256 암호화 키 (32바이트)
```

**특징**:
- ✅ 사전 공격에 강함
- ✅ GPU 병렬 처리 방어
- ✅ 브루트포스 공격 어려움

#### 암호화 파일 구조

```
[메타데이터 길이 (4 bytes)]
    ↓
[메타데이터 (JSON)]
    ├─ version: 1.0
    ├─ algorithm: aes-256-gcm
    ├─ encryptionMethod: password
    ├─ timestamp: 2026-04-26T00:00:00Z
    ├─ originalSize: 52428800
    ├─ salt: (16 bytes, hex)
    └─ iv: (12 bytes, hex)
    ↓
[암호화 데이터]
    ├─ IV (12 bytes, prepended)
    ├─ Salt (16 bytes, prepended)
    ├─ AuthTag (16 bytes, for GCM)
    └─ 실제 암호화된 tar.gz 파일
```

### 4. 복호화 검증

```
API 호출 시:
├─ 메타데이터 읽기
├─ 암호화 방식 확인
├─ 비밀번호 입력 요구
├─ PBKDF2로 키 도출
├─ AES-GCM으로 복호화
├─ AuthTag 검증 (무결성 확인)
└─ 성공/실패 응답
```

## 📊 성능 영향

### 암호화로 인한 시간 증가

```
작업                   | 시간 (без암호화) | 시간 (암호화) | 증가량
──────────────────────┼─────────────────┼──────────────┼────────
백업 생성 (50MB)      | 5-10초           | 7-12초       | +2초
복원 (50MB)           | 8-15초           | 10-17초      | +2-3초
USB 전송              | 변화 없음         | 변화 없음     | 0초
```

### 용량 영향

```
파일                  | 크기
──────────────────────┼──────────
tar.gz (압축)         | ~50MB
.tar.gz.enc (암호화)  | ~51MB (+1-2%)
메타데이터 오버헤드   | ~0.5KB
```

**결론**: 암호화는 거의 영향을 주지 않음 ✅

## 🆘 문제 해결

### "복호화 실패: 비밀번호가 올바르지 않습니다"

**원인**: 잘못된 비밀번호 또는 파일 손상

**해결책**:
1. 비밀번호 정확성 재확인
   ```bash
   # 백업 메타데이터 확인
   file kinder-backup-2026-04-26.tar.gz.enc
   ```

2. 파일 무결성 확인
   ```bash
   # 파일 크기, 수정 시간 확인
   ls -lh kinder-backup-2026-04-26.tar.gz.enc
   ```

3. USB 복사 시 손상 여부 확인
   ```bash
   # 파일 해시 비교
   sha256sum kinder-backup-2026-04-26.tar.gz.enc
   ```

### "암호화 작업 중단됨"

**원인**: BACKUP_PASSWORD 환경 변수 누락

**해결책**:
```bash
# .env.local 확인
cat .env.local | grep BACKUP_PASSWORD

# 없으면 추가
echo "BACKUP_PASSWORD=YourStrongPassword123!@#" >> .env.local

# 앱 재시작
npm run dev
```

### "비밀번호 입력 폼이 나타나지 않음"

**원인**: 암호화 파일이 아님 또는 네트워크 오류

**해결책**:
```bash
# 파일 확인
ls -la *.tar.gz.enc

# 메타데이터 조회
curl http://localhost:3000/api/admin/backup/status
```
