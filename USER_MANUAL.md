## 개요

**AKMS(ABA Child Management System)**는 응용행동분석(ABA) 기반의 아동 관리 시스템입니다. 

- **대상**: 치료사, 부모, 교사
- **목적**: 아동의 발달 진행 상황을 추적하고 맞춤형 교육 계획을 수립
- **기반**: 행동 분석 및 14개 발달 영역 기반 커리큘럼

### 주요 특징
- 🔐 **강화된 보안**: 디바이스 및 사용자 승인 시스템
- 📊 **상세 데이터 분석**: ABC(선행사건-행동-결과) 기반 기록
- 👶 **맞춤형 커리큘럼**: 14개 도메인 × 252개 LTO 구조
- 📈 **시각화 대시보드**: 실시간 진행 상황 모니터링

## 시스템 보안 개요

### 보안 정책

AKMS는 다음과 같은 보안 정책을 적용합니다:

#### 1. 디바이스 인증
- **처음 로그인 시** 사용하는 기기(컴퓨터, 태블릿 등)를 자동으로 등록합니다
- 각 디바이스는 고유한 **디바이스 ID**를 부여받습니다
- 디바이스 ID는 다음 정보를 바탕으로 생성됩니다:
  - 브라우저 정보 (User Agent)
  - 화면 해상도
  - 시스템 타임존
  - IP 주소

#### 2. 사용자 승인
- **모든 신규 사용자**는 관리자의 승인이 필요합니다
- 승인 전에는 시스템에 접근할 수 없습니다
- 관리자는 다음 정보를 확인한 후 승인합니다:
  - 사용자 이메일
  - 디바이스 정보
  - IP 주소
  - 등록 시간

#### 3. 접근 제어
- **승인된 디바이스에서만** 로그인 가능합니다
- 각 사용자의 접근은 **접근 로그**에 기록됩니다
- 거부된 접근 시도도 모두 기록됩니다

#### 4. 세션 관리
- 로그아웃 시 세션 정보가 삭제됩니다
- 브라우저를 닫으면 자동으로 로그아웃됩니다

## 로그인 절차

### 첫 로그인 (신규 사용자)

#### 단계 1: 로그인 화면 접속
1. 웹 브라우저에서 AKMS에 접속합니다
2. 로그인 화면이 표시됩니다

#### 단계 2: 자격증명 입력
```
이메일: admin@akms.com
비밀번호: admin123
```
- 테스트용 관리자 계정 위 정보를 사용합니다
- 실제 운영 시에는 다른 계정을 사용하게 됩니다

#### 단계 3: 디바이스 등록
1. "로그인" 버튼을 클릭합니다
2. 자동으로 다음 정보가 수집됩니다:
   - **디바이스 ID**: 이 기기의 고유 식별자
   - **IP 주소**: 현재 인터넷 연결의 IP 주소
   - **브라우저 정보**: 사용 중인 브라우저 종류 및 버전
   - **기기명**: Windows/Mac/기타 플랫폼 정보

#### 단계 4: 승인 대기
- "승인 대기 중" 화면이 표시됩니다
- 다음 정보가 관리자에게 전송됩니다:
  - 사용자 이메일
  - 디바이스 ID (처음 8글자)
  - IP 주소
  - 등록 시간

**⏳ 처리 시간**: 관리자 확인 후 승인 (보통 즉시 ~ 몇 분)

### 기존 사용자 로그인

1. 로그인 화면에서 이메일과 비밀번호를 입력합니다
2. 승인된 디바이스에서는 **자동으로 로그인**됩니다
3. 다양한 디바이스에서 로그인하려면:
   - 각 기기에서 처음 로그인할 때 등록 및 승인 필요
   - 승인 후에는 해당 기기에서 자유롭게 로그인 가능

## 디바이스 등록 및 승인

### 디바이스란?

- 시스템에 접근하는 **특정 컴퓨터 또는 태블릿**
- 각 디바이스는 고유한 ID를 가짐
- 승인된 디바이스만 사용 가능

### 예시

| 상황 | 필요한 작업 |
|------|-----------|
| 같은 컴퓨터에서 계속 사용 | 첫 로그인만 승인 필요 |
| 다른 컴퓨터에서 로그인 | 새 디바이스로 등록 및 승인 필요 |
| 태블릿을 추가로 사용 | 태블릿 등록 및 승인 필요 |
| 브라우저를 변경 | 선택 사항 (같은 컴퓨터면 재승인 불필요) |

### 디바이스 등록 화면

로그인 시 다음 정보가 표시됩니다:

```
📱 디바이스 정보
• 디바이스 ID: a1b2c3d4...
• IP 주소: 123.456.789.012
• 브라우저: Mozilla/5.0...

⏳ 상태: 승인 대기 중

관리자가 위 정보를 확인한 후 승인하면
시스템에 접근할 수 있습니다.
```

## 관리자 승인 인터페이스

### 승인 페이지 접속

**관리자만 접속 가능합니다**

1. 로그인 후 왼쪽 사이드바에서 **"👨‍💼 승인관리"** 클릭
2. 또는 URL에서 `/admin/approvals` 접속

### 승인 페이지 구성

#### 1. 필터 탭
```
⏳ 대기중 | ✅ 승인됨 | 📋 전체
```
- **대기중**: 아직 승인하지 않은 항목
- **승인됨**: 이미 승인한 항목
- **전체**: 모든 항목

#### 2. 사용자 승인 섹션

각 사용자에 대해 다음 정보가 표시됩니다:

```
📧 이메일: user@example.com
👤 이름: 김철수 (역할: admin)

등록일: 2024년 4월 26일 14:30

[✅ 승인]  [❌ 거부]
```

**승인 시 결과**:
- 사용자 상태: "대기중" → "활성"
- 사용자 승인됨: 표시
- 해당 사용자가 로그인 가능

**거부 시 결과**:
- 사용자 상태: "중단"
- 로그인 불가능
- 재요청 필요

#### 3. 디바이스 승인 섹션

각 디바이스에 대해 다음 정보가 표시됩니다:

```
🖥️ 디바이스 ID: a1b2c3d4e5f6...
기기명: Windows - 2024.4.26

IP 주소: 123.456.789.012
등록일: 2024년 4월 26일 14:30

브라우저: Mozilla/5.0 (Windows NT 10.0; Win64; x64)...

[✅ 승인]  [❌ 거부]
```

**승인 시 결과**:
- 디바이스 상태: "대기중" → "승인됨"
- 승인 일시 기록
- 사용자가 이 디바이스에서 로그인 가능

**거부 시 결과**:
- 디바이스가 목록에서 제거됨
- 다시 로그인 시 새로 등록 필요

### 승인 프로세스 예시

#### 시나리오: 새로운 치료사가 시작

1. **등록 단계**
   - 치료사가 AKMS에 처음 접속
   - 이메일: therapist@akms.com
   - 디바이스 1 (병원 컴퓨터)

2. **대기 단계**
   - 승인 대기 화면 표시
   - 관리자 알림 (자동)

3. **승인 단계**
   - 관리자가 승인 관리 페이지 열기
   - 새 사용자 확인
   - 이메일과 IP 주소 검증
   - "✅ 승인" 클릭

4. **접근 단계**
   - 치료사가 시스템 접근 가능
   - 다양한 기능 사용 가능

### 승인 시 확인 사항

승인 전에 다음을 확인하세요:

- [ ] 이메일이 올바른가?
- [ ] 예상되는 IP 주소인가?
- [ ] 등록 시간이 합리적인가?
- [ ] 디바이스 정보가 알려진 기기인가?

## 커리큘럼 구조

### 14개 발달 영역

| # | 영역 | 설명 | LTO 수 |
|---|------|------|--------|
| 1 | 요청(Mand) | 아동이 원하는 것을 요청하는 언어 기술 | 18 |
| 2 | 명명(Tact) | 환경의 자극에 대해 명명하는 언어 기술 | 18 |
| 3 | 상호언어(Intraverbal) | 타인의 언어 자극에 반응하여 언어하는 기술 | 18 |
| 4 | 따라말하기(Echoic) | 타인의 언어를 따라하는 기술 | 18 |
| 5 | 수용언어(Receptive) | 언어를 이해하고 반응하는 기술 | 18 |
| 6 | 사회성(Social) | 다른 사람과의 상호작용 및 의사소통 기술 | 18 |
| 7 | 소근육발달(Fine Motor) | 손가락, 손, 팔의 정교한 움직임 발달 | 18 |
| 8 | 대근육발달(Gross Motor) | 머리, 몸통, 팔, 다리의 큰 근육 발달 | 18 |
| 9 | 놀이(Play Skills) | 혼자 또는 타인과 놀이하는 기술 | 18 |
| 10 | 학업기초(Academic Skills) | 읽기, 쓰기, 수학 등 학업 기초 기술 | 18 |
| 11 | 생활기술(Self-Care) | 식사, 화장실 사용, 옷 입기 등 자조 기술 | 18 |
| 12 | 행동관리(Behavior Management) | 행동 제어 및 자기조절 기술 | 18 |
| 13 | 지시따르기(Compliance) | 성인의 지시를 따르는 기술 | 18 |
| 14 | 주의집중(Attention) | 특정 자극에 주의를 기울이는 기술 | 18 |

**총합**: 14 × 18 = 252 LTO

### 계층 구조

각 영역은 다음과 같이 구성됩니다:

```
영역 (Domain)
  └─ 장기목표 (LTO: Long-Term Objective)
       └─ 단기목표 (STO: Short-Term Objective)
            └─ 1. 보조 있이 시도
            └─ 2. 부분적 독립
            └─ 3. 독립적 수행
            └─ 4. 일반화
```

### 예시: "요청(Mand)" 영역

```
영역: 요청(Mand)
  LTO 1: 일반적인 물건 요청하기
    목표: 단어나 문장으로 원하는 물건을 요청
    교수 팁:
    • 보조 계층: 모델링 → 신체적 보조 → 언어적 힌트
    • 강화: 행동 직후 즉시 강화
    • 일반화: 다양한 환경에서 훈련
    
    STO 1: 보조 있이 시도
      설명: 성인의 신체적/언어적 보조로 요청 시도
    
    STO 2: 부분적 독립
      설명: 최소한의 보조로 요청 수행
    
    STO 3: 독립적 수행
      설명: 보조 없이 독립적으로 요청
    
    STO 4: 일반화
      설명: 다양한 상황에서 요청 적용
```

## 데이터 기록

### ABC 데이터 기록이란?

**ABC 데이터**는 행동 분석의 핵심입니다:

- **A (Antecedent)**: 선행사건 - 행동 발생 전의 상황
- **B (Behavior)**: 행동 - 아동이 보인 행동
- **C (Consequence)**: 결과 - 행동 후의 결과

### 예시

| 영역 | 설명 | 예시 |
|------|------|------|
| A | 선행사건 | 성인이 "물 가져와"라고 지시 |
| B | 행동 | 아동이 물을 가져옴 |
| C | 결과 | 성인이 "잘했어! 우와~" 칭찬 |

### 기록 방법

1. **데이터 기록 페이지** 접속
2. **아동 및 LTO/STO 선택**
3. **세션 정보 입력**:
   - 시작 시간 / 종료 시간
   - 점수 (0-100)
   - 특이 사항 기록

4. **ABC 데이터 입력** (선택사항):
   - 선행사건: 어떤 자극이 있었는가?
   - 행동: 아동이 어떻게 반응했는가?
   - 결과: 그 후 어떻게 되었는가?

5. **저장**

### 효과적인 기록 팁

- ✏️ **구체적으로**: "잘했어" X → "9번 시도 중 8번 성공" O
- ⏰ **즉시 기록**: 세션 직후 기록하기
- 📊 **일관성**: 매일 같은 시간에 기록
- 🎯 **목표 중심**: 목표와 관련된 행동만 기록

## FAQ

### Q1: 다른 컴퓨터에서 로그인할 수 없습니다

**A:** 각 컴퓨터는 별도의 디바이스로 등록됩니다. 처음 사용하는 컴퓨터에서는:

1. 로그인 시도
2. 승인 대기 화면 나타남
3. 관리자에게 알림이 전송됨
4. 관리자가 승인
5. 로그인 가능

### Q2: 승인이 얼마나 걸리나요?

**A:** 일반적으로 **즉시 ~ 5분**입니다. 관리자가 접속 후 승인 관리 페이지에서 "승인" 버튼을 클릭하면 됩니다.

### Q3: 승인을 거부받으면 어떻게 되나요?

**A:** 거부된 경우:
- 해당 계정으로 로그인 불가능
- 관리자에게 재요청 필요
- 새 이메일로 다시 신청 가능

### Q4: 디바이스를 변경해야 하는 경우

**A:** 다음 상황에서 새 디바이스 등록이 필요합니다:

- 다른 컴퓨터 사용 ✅
- 다른 태블릿 사용 ✅
- 같은 컴퓨터, 다른 브라우저 (선택사항)
- 다른 네트워크 사용 (같은 컴퓨터면 재승인 불필요)

### Q5: 승인 후 로그인할 수 없습니다

**A:** 다음을 확인하세요:

1. 브라우저 캐시 삭제: Ctrl+Shift+Delete
2. 새로고침: F5
3. 다시 로그인 시도
4. 여전히 안 되면 관리자에게 연락

### Q6: 계정을 여러 개 만들 수 있나요?

**A:** 각 사용자는 **한 개의 계정**을 가져야 합니다. 같은 사람이 여러 계정을 만들 수는 있지만 권장되지 않습니다. 필요한 경우 관리자에게 문의하세요.

### Q7: 접근 로그는 어디서 확인하나요?

**A:** 접근 로그는:
- 관리자만 확인 가능
- 시스템 설정 메뉴에서 조회
- 모든 로그인/로그아웃 기록 포함

### Q8: 비밀번호를 잊었습니다

**A:** 현재 비밀번호 재설정 기능은 없습니다. 관리자에게 연락하여 계정을 초기화해주도록 요청하세요.
